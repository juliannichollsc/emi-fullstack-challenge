name: a11y
description: Accessibility floor the jest-tester relies on for RTL queries.
metadata:
  type: feedback
---

Every interactive element must be:
- Keyboard reachable and operable.
- Labeled (`<label htmlFor>` or `aria-label`).
- Discoverable by `getByRole` with an accessible name — the jest-tester depends on this.
- Contrasted at WCAG AA or better.

**Why:** RTL tests use role+name queries. If the UI agent uses `<div onClick>` and unlabeled icons, tests break and we lose marks for senior accessibility.

**How to apply:** prefer semantic HTML (`<button>`, `<form>`, `<nav>`) over generic divs. Add `aria-label` when text alone isn't a discoverable name.
